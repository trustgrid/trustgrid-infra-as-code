## Codebase Patterns
- GCP examples use GitHub source refs pinned to `v0.10.0` (latest released tag); all module sources must use `?ref=v0.10.0` ŌĆö v0.11.0 does not exist, never use non-existent future tags
- trustgrid/tg provider config: uses `api_host`, `api_key_id`, `api_key_secret`, `org_id` fields (NOT `host`/`org`) ŌĆö credentials supplied via TG_API_KEY_ID and TG_API_KEY_SECRET env vars
- tg_license resource outputs: `license` (JWT), `uid`, `fqdn`, `id` ŌĆö pass `license` to compute module, `uid` to data.tg_node for readiness gate
- tg_network_config: use `node_id` for per-node config, `cluster_fqdn` for cluster-level config ŌĆö NOT `node`/`cluster` field names
- tg_node_cluster_config: uses `node_id` field (not `node`) ŌĆö fields: node_id, host, port, enabled (optional: status_host, status_port)
- tg_cluster_member: uses `cluster_fqdn` and `node_id` fields
- data.tg_node: accepts `uid` (from tg_license.*.uid), `timeout` (seconds), and `fqdn` ŌĆö blocks until node is online
- Stale `.terraform.lock.hcl` files with non-normalized constraint format (`>= 6.0` vs `>= 6.0.0`) block `terraform init` ŌĆö delete and re-init when switching module sources from GitHub refs to relative paths
- HA gateway heartbeat firewall (TCP 9000) is a direct `google_compute_firewall` resource in the example, NOT a module ŌĆö source_ranges are the two data subnet CIDRs, target is the gateway network tag
- Network firewall modules live under `gcp/terraform/modules/network/`; compute modules under `gcp/terraform/modules/compute/`; IAM modules under `gcp/terraform/modules/iam/`
- IAM service account module outputs a `member` output pre-formatted as `serviceAccount:<email>` for direct use in `google_project_iam_binding.members`
- `google_project_iam_binding` is AUTHORITATIVE for its role ŌĆö use it when you own the full membership of a custom role; use `google_project_iam_member` for non-authoritative (additive) bindings
- GCP route failover requires exactly 4 permissions: `compute.routes.{list,get,create,delete}` ŌĆö routes are project-global so the IAM binding must be at project scope
- Custom role IDs: `can(regex("^[a-zA-Z0-9_.]{1,64}$", var.role_id))` is the correct validation pattern
- `log_config { metadata = var.enable_logging ? "INCLUDE_ALL_METADATA" : "EXCLUDE_ALL_METADATA" }` is the correct pattern for optional GCP firewall logging
- GCP firewall rule port strings must be quoted even when using variable interpolation: `ports = ["${var.gateway_port}"]`
- GCP modules use `~> 6.0` provider constraint (same tilde-range style as Azure/ThousandEyes)
- Bootstrap templates live in `<module>/templates/bootstrap.sh.tpl`, wired via `templatefile()` into a `locals {}` block, then consumed by `metadata_startup_script` on the instance resource
- Sensitive variables (license, registration_key) are coerced from `null` to `""` before passing to templatefile(): `var.license != null ? var.license : ""`
- Always inject module-managed metadata keys (tg-license, tg-registration-key) as conditional ternary expressions in `merge()` ŌĆö never pass nulls into the metadata map
- `lifecycle { ignore_changes = all }` on every VM/instance resource
- `set -euo pipefail` mandatory on every `.sh.tpl` template
- TF_DOCS markers must be kept in sync with variables/outputs when adding new variables
- For conditional GCP static IP resources use `count = var.mode == "x" ? 1 : 0` and reference via `[0]`; region is derived from zone by dropping last segment: `join("-", slice(split("-", var.zone), 0, length(split("-", var.zone)) - 1))`
- `dynamic "access_config"` block on `network_interface` is the correct pattern for optional external IP assignment; `for_each = local.external_ip != null ? [local.external_ip] : []`
- Image selection two-tier priority: (1) `image_name` set ŌåÆ used directly, project/family ignored; (2) `image_name = null` ŌåÆ `data.google_compute_image` resolves latest from `image_project`+`image_family`
- Guard `image_name` against empty-string misuse with: `condition = var.image_name == null || length(trimspace(var.image_name)) > 0`
- Guard `image_project` / `image_family` against empty strings with: `condition = length(trimspace(var.image_X)) > 0`
