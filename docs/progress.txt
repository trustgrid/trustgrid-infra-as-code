## Codebase Patterns
- GCP modules use `~> 6.0` provider constraint (same tilde-range style as Azure/ThousandEyes)
- Bootstrap templates live in `<module>/templates/bootstrap.sh.tpl`, wired via `templatefile()` into a `locals {}` block, then consumed by `metadata_startup_script` on the instance resource
- Sensitive variables (license, registration_key) are coerced from `null` to `""` before passing to templatefile(): `var.license != null ? var.license : ""`
- Always inject module-managed metadata keys (tg-license, tg-registration-key) as conditional ternary expressions in `merge()` — never pass nulls into the metadata map
- `lifecycle { ignore_changes = all }` on every VM/instance resource
- `set -euo pipefail` mandatory on every `.sh.tpl` template
- TF_DOCS markers must be kept in sync with variables/outputs when adding new variables
- For conditional GCP static IP resources use `count = var.mode == "x" ? 1 : 0` and reference via `[0]`; region is derived from zone by dropping last segment: `join("-", slice(split("-", var.zone), 0, length(split("-", var.zone)) - 1))`
- `dynamic "access_config"` block on `network_interface` is the correct pattern for optional external IP assignment; `for_each = local.external_ip != null ? [local.external_ip] : []`
